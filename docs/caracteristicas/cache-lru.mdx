# Sistema de Caché LRU

SIAA implementa un caché LRU (Least Recently Used) con protección de hilos que almacena respuestas completas a consultas documentales frecuentes. El caché puede entregar respuestas en ~5ms frente a los ~25s de una consulta no cacheada — una **aceleración de hasta 8.800×**.

---

## Configuración

```python
CACHE_MAX_ENTRADAS = 200    # Entradas máximas (~1 MB en RAM)
CACHE_TTL_SEGUNDOS = 3600   # 1 hora por entrada
CACHE_SOLO_DOC     = True   # Solo cachear consultas documentales
```

---

## Generación de Clave de Caché

La clave se genera a partir de una versión normalizada de la pregunta, insensible a tildes, puntuación y mayúsculas:

```python
def _clave_cache(texto: str) -> str:
    t = texto.lower()
    t = re.sub(r'[^\w\s]', '', t)       # Eliminar puntuación
    t = unicodedata.normalize('NFD', t)  # Separar diacríticos
    t = ''.join(c for c in t if unicodedata.category(c) != 'Mn')  # Quitar tildes
    t = re.sub(r'\s+', ' ', t).strip()
    return hashlib.sha256(t.encode()).hexdigest()[:16]
```

**Todas estas variaciones producen la misma clave:**

```
"¿Cuándo debo reportar?"
"CUÁNDO DEBO REPORTAR"
"cuando debo reportar"
"¿¿¿Cuándo... debo reportar???"
```

→ Clave: `a7f3c8e9b2d14f56`

## Reglas de Almacenamiento

| Condición | ¿Se cachea? |
|-----------|-------------|
| Respuesta de consulta documental | ✅ Sí |
| Respuesta vacía | ❌ No |
| Respuesta "no encontré esa información" | ❌ No — puede cambiar al actualizar docs |
| Saludos y conversación general | ❌ No (controlado por `CACHE_SOLO_DOC`) |

---

## Impacto en Rendimiento

| Escenario | Tiempo | Recursos |
|-----------|--------|----------|
| Primera consulta "¿Cuándo reportar?" | ~30s | Enrutador + chunks + modelo |
| 2ª a 26ª consulta idéntica | ~5ms | Solo RAM |

**Caso real — 26 despachos con la misma pregunta:**

- Consultas 2–26: 25 × 38s ahorrados = **950 segundos** (15.8 minutos)
- Hit rate estimado: 30–40%

```bash
# Ver estadísticas

# Limpiar caché (después de actualizar documentos)
curl -X DELETE http://localhost:5000/siaa/cache

# Flujo recomendado al actualizar documentos:
python3 convertidor.py
curl -X DELETE http://localhost:5000/siaa/cache
curl http://localhost:5000/siaa/recargar
```

## Entrega de Respuestas Cacheadas

Las respuestas del caché se entregan vía SSE con streaming simulado para mantener la compatibilidad con el frontend:

```python
# Simula el efecto de "escritura" token a token
chunk_size = 40  # chars por "token"
for i in range(0, len(respuesta_cached), chunk_size):
    trozo = respuesta_cached[i:i+chunk_size]
    yield f'data: {{"choices":[{{"delta":{{"content":"{safe}"}}}}]}}\n\n'
yield "data: [DONE]\n\n"
```

Las respuestas cacheadas incluyen el encabezado `X-Cache: HIT`.
